## ADDED Requirements

### Requirement: 显示 Git 提交历史列表
系统 SHALL 在右侧区域显示 Git 仓库的提交历史列表，以便用户查看代码变更历史。

#### Scenario: 初始加载最近 20 条提交
- **WHEN** 用户点击 Git 仓库节点
- **THEN** 系统应在右侧区域显示提交历史列表
- **AND** 初始加载最近 20 条提交记录
- **AND** 提交按时间倒序排列（最新的在最上面）

#### Scenario: 显示提交基本信息
- **WHEN** 系统显示提交列表
- **THEN** 每个提交应显示以下信息：
  - 提交 SHA（前 8 位）
  - 提交消息（完整消息）
  - 作者名称
  - 提交时间（相对时间格式）
  - 变更文件数量

#### Scenario: 提交列表使用时间线布局
- **WHEN** 系统显示提交列表
- **THEN** 应使用时间线（Timeline）组件展示
- **AND** 时间线应垂直排列
- **AND** 每个提交应占据时间线的一个节点

### Requirement: 提交历史支持分页加载
系统 SHALL 支持分页加载更多提交历史，以避免一次性加载过多数据导致性能问题。

#### Scenario: 提供"加载更多"按钮
- **WHEN** 用户查看提交历史列表
- **AND** 已加载的提交数量少于总提交数
- **THEN** 系统应在列表底部显示"加载更多"按钮
- **AND** 按钮应在滚动到底部时自动出现或始终显示

#### Scenario: 点击"加载更多"加载下一页
- **WHEN** 用户点击"加载更多"按钮
- **THEN** 系统应加载下一页 20 条提交记录
- **AND** 新加载的提交应追加到现有列表末尾
- **AND** 加载过程中应显示加载状态

#### Scenario: 没有更多提交时隐藏按钮
- **WHEN** 用户查看提交历史列表
- **AND** 已加载所有提交记录
- **THEN** 系统应隐藏"加载更多"按钮
- **AND** 可选显示"已加载全部提交"提示信息

#### Scenario: 限制单次加载的提交数量
- **WHEN** 系统加载提交历史
- **THEN** 每次加载的提交数量不应超过 20 条
- **AND** 后端应强制执行此限制
- **AND** 前端应通过分页参数控制加载范围

### Requirement: 提交历史支持提交详情展示
系统 SHALL 支持展示单个提交的详细信息，包括变更的文件列表。

#### Scenario: 点击提交展开详情
- **WHEN** 用户点击提交列表中的某个提交
- **THEN** 系统应展开该提交的详细信息
- **AND** 详细信息应包括：
  - 完整的提交 SHA（40 位）
  - 作者邮箱
  - 提交时间（绝对时间格式）
  - 变更的文件列表

#### Scenario: 显示变更文件列表
- **WHEN** 用户查看提交详情
- **THEN** 系统应显示该提交变更的所有文件
- **AND** 每个文件应显示文件名和变更类型（新增、修改、删除）
- **AND** 变更类型应使用不同颜色或图标区分：
  - 新增文件：绿色
  - 修改文件：蓝色
  - 删除文件：红色

#### Scenario: 再次点击提交折叠详情
- **WHEN** 用户点击已展开的提交
- **THEN** 系统应折叠该提交的详细信息
- **AND** 只显示基本信息（SHA、消息、作者、时间）

### Requirement: 提交历史支持搜索和过滤
系统 SHALL 支持根据提交消息、作者或 SHA 搜索和过滤提交记录。

#### Scenario: 提供搜索框
- **WHEN** 用户查看提交历史区域
- **THEN** 系统应在区域顶部显示搜索框
- **AND** 搜索框应支持输入提交消息、作者名称或 SHA

#### Scenario: 实时搜索过滤
- **WHEN** 用户在搜索框中输入关键词
- **THEN** 系统应实时过滤已加载的提交列表
- **AND** 只显示匹配的提交记录
- **AND** 匹配规则包括：
  - 提交消息包含关键词
  - 作者名称包含关键词
  - SHA 包含关键词

#### Scenario: 清空搜索恢复列表
- **WHEN** 用户清空搜索框内容
- **THEN** 系统应恢复显示所有已加载的提交记录
- **AND** 列表顺序应保持原始的时间倒序

#### Scenario: 搜索无结果显示提示
- **WHEN** 用户搜索的关键词没有匹配的提交
- **THEN** 系统应显示"未找到匹配的提交"提示信息
- **AND** 提示信息应清晰友好

### Requirement: 提交历史支持复制提交 SHA
系统 SHALL 允许用户快速复制提交的完整 SHA，以便在其他工具中使用。

#### Scenario: 点击提交 SHA 复制到剪贴板
- **WHEN** 用户点击提交列表中的 SHA（前 8 位）
- **THEN** 系统应复制完整的 40 位 SHA 到剪贴板
- **AND** 系统应显示"已复制"提示信息
- **AND** 提示信息应在 2 秒后自动消失

#### Scenario: 点击完整 SHA 复制到剪贴板
- **WHEN** 用户查看提交详情
- **AND** 点击完整的 40 位 SHA
- **THEN** 系统应复制完整的 SHA 到剪贴板
- **AND** 系统应显示"已复制"提示信息

### Requirement: 提交历史使用卡片式布局
系统 SHALL 将提交历史展示在独立的卡片组件中，以与其他内容区域区分。

#### Scenario: 提交历史卡片标题
- **WHEN** 用户查看提交历史区域
- **THEN** 系统应显示卡片标题，如"提交历史"或"Commit History"
- **AND** 标题应清晰明确，字体稍大

#### Scenario: 提交历史卡片样式
- **WHEN** 用户查看提交历史区域
- **THEN** 卡片应具有轻微的阴影和圆角
- **AND** 卡片背景色应为白色或浅灰色
- **AND** 卡片应有适当的内边距（padding）

#### Scenario: 提交历史仅在 Git 仓库中显示
- **WHEN** 用户点击非 Git 仓库的文件夹或文件节点
- **THEN** 系统不应显示提交历史卡片
- **AND** 右侧区域应显示其他相关内容

### Requirement: 提交历史加载状态反馈
系统 SHALL 在加载提交历史时提供加载状态反馈，以改善用户体验。

#### Scenario: 显示初始加载状态
- **WHEN** 用户点击 Git 仓库节点
- **AND** 提交历史正在加载中
- **THEN** 系统应在提交历史区域显示加载指示器（如 loading spinner）
- **AND** 加载指示器应居中显示

#### Scenario: 显示加载更多状态
- **WHEN** 用户点击"加载更多"按钮
- **AND** 更多提交正在加载中
- **THEN** 系统应在按钮上显示加载状态
- **AND** 按钮应变为禁用状态，防止重复点击

#### Scenario: 加载失败显示错误信息
- **WHEN** 系统尝试加载提交历史
- **AND** 加载过程失败
- **THEN** 系统应显示错误提示信息
- **AND** 错误信息应说明失败原因
- **AND** 错误信息应提供重试按钮

### Requirement: 提交历史支持刷新操作
系统 SHALL 允许用户手动刷新提交历史，以获取最新的提交记录。

#### Scenario: 提供刷新按钮
- **WHEN** 用户查看提交历史区域
- **THEN** 系统应在卡片右上角显示刷新按钮
- **AND** 刷新按钮应使用刷新图标

#### Scenario: 点击刷新按钮重新加载
- **WHEN** 用户点击刷新按钮
- **THEN** 系统应清空当前提交列表
- **AND** 重新从后端加载最新的提交记录
- **AND** 刷新过程中应显示加载状态
- **AND** 刷新完成后应更新显示的提交列表

### Requirement: 提交历史显示相对时间
系统 SHALL 使用相对时间格式显示提交时间，以便用户快速了解提交的新旧程度。

#### Scenario: 显示最近的提交相对时间
- **WHEN** 提交时间在 24 小时内
- **THEN** 系统应显示相对时间（如"2 小时前"、"30 分钟前"）
- **AND** 时间应根据当前时间动态计算

#### Scenario: 显示较旧的提交相对时间
- **WHEN** 提交时间超过 24 小时但在 30 天内
- **THEN** 系统应显示相对时间（如"3 天前"、"1 周前"）

#### Scenario: 显示很久以前的提交绝对时间
- **WHEN** 提交时间超过 30 天
- **THEN** 系统应显示绝对时间（如"2025-03-15"）
- **AND** 日期格式应为 YYYY-MM-DD

#### Scenario: 鼠标悬停显示绝对时间
- **WHEN** 用户将鼠标悬停在相对时间上
- **THEN** 系统应显示工具提示（tooltip）
- **AND** 工具提示应显示完整的绝对时间（如"2025-04-29 14:30:00"）
