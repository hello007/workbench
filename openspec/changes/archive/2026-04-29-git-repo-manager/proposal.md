## Why

当前缺乏一个便捷的可视化工具来统一管理本地Git仓库和文件系统。开发者需要在命令行和文件管理器之间频繁切换，操作繁琐且效率低下。特别是对于多个项目的管理，缺乏统一的界面来执行常见的Git操作（克隆、拉取、查看历史）和文件管理任务。

## What Changes

开发一个基于Wails的桌面Git仓库管理工具，提供以下核心功能：

- **工作目录管理**：支持配置多个常用工作目录，通过下拉框快速切换，配置持久化存储到本地JSON文件
- **可视化文件树**：左侧展示目录结构，支持懒加载、展开/收起，自动过滤.git隐藏文件夹
- **文件操作**：支持文件夹和文件的创建、删除、重命名，以及文本文件内容预览（只读）
- **Git仓库集成**：
  - 自动检测Git仓库并标识
  - 显示仓库信息（分支、远程、URL）
  - 支持克隆操作（输入GitHub URL克隆到选中目录，含冲突检测）
  - 支持拉取更新操作
  - 提交历史查看（分页展示）
- **文件树控制**：提供全部展开/全部收起功能
- **桌面应用部署**：使用Wails框架打包为原生Windows桌面应用，单一可执行exe文件（15-20MB），独立窗口运行，无需任何运行时依赖

## Capabilities

### New Capabilities

- `directory-management`: 工作目录的配置管理，包括增删改查和JSON持久化存储
- `file-tree-browser`: 文件系统的可视化浏览，支持懒加载、展开/收起、Git仓库检测
- `file-operations`: 文件和文件夹的基础操作，包括创建、删除、重命名、预览
- `git-integration`: Git仓库的集成操作，包括信息展示、克隆、拉取、提交历史查看

### Modified Capabilities

无（这是一个全新的项目）

## Impact

- **技术栈**：引入Wails（桌面应用框架）+ Go（后端逻辑）+ Vue3 + Element Plus（前端UI）技术栈
- **依赖项**：
  - Wails v2.5+ 桌面应用框架
  - 后端使用Go标准库os/exec执行系统Git命令
  - 后端方法通过Wails绑定到前端，无需HTTP API
  - 前端使用Wails运行时与Go通信
  - 前端需要Element Plus UI组件库
- **部署方式**：Wails打包为原生桌面应用，前端资源嵌入二进制文件，编译为单一可执行文件
- **系统要求**：
  - 需要预安装Git（工具不包含Git本身）
  - Windows 10/11（WebView2运行时，系统自带）
  - 支持Windows文件系统路径
  - 需要文件系统读写权限
  - **无需任何运行时环境**（编译后的exe可直接运行）
- **架构设计**：前后端在同一进程中，通过Wails绑定通信，无需HTTP服务器
- **数据存储**：使用本地JSON文件存储工作目录配置，无需数据库
- **性能优势**：
  - 编译后单一exe文件（15-20MB）
  - 启动速度快（0.5-1秒）
  - 内存占用低（30-50MB）
  - 原生桌面应用体验
  - 无需用户安装JVM、Node.js或其他运行时
- **用户体验提升**：
  - 独立窗口，不是浏览器标签
  - 系统托盘支持（可选）
  - 原生菜单和右键菜单
  - 应用图标和任务栏集成
  - 更像专业开发工具（类似VS Code）
