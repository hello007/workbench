## ADDED Requirements

### Requirement: 文件树懒加载
系统应支持按需加载文件树节点，避免一次性加载大量文件。

#### Scenario: 初始加载根目录
- **WHEN** 用户选择一个工作目录
- **THEN** 系统仅加载根目录的直接子节点（一层深度）
- **AND** 系统返回子节点列表，包含文件和文件夹

#### Scenario: 展开文件夹加载子节点
- **WHEN** 用户点击文件夹的展开按钮
- **THEN** 系统加载该文件夹的直接子节点
- **AND** 系统将子节点添加到文件树中
- **AND** 文件夹显示为展开状态

#### Scenario: 收起文件夹
- **WHEN** 用户点击文件夹的收起按钮
- **THEN** 系统收起该文件夹的子节点显示
- **AND** 文件夹显示为折叠状态
- **AND** 已加载的子节点保留在内存中

#### Scenario: 加载空文件夹
- **WHEN** 用户展开一个空文件夹
- **THEN** 系统返回空子节点列表
- **AND** 文件夹不显示展开/收起按钮

### Requirement: Git仓库检测
系统应自动检测文件夹是否为Git仓库根目录。

#### Scenario: 检测到Git仓库
- **WHEN** 文件夹包含.git子目录
- **THEN** 系统在文件树中标记该文件夹为Git仓库
- **AND** 显示Git仓库图标（如 ✓ 标识）

#### Scenario: 非Git仓库文件夹
- **WHEN** 文件夹不包含.git子目录
- **THEN** 系统将该文件夹标记为普通文件夹
- **AND** 显示普通文件夹图标

#### Scenario: Git仓库子目录
- **WHEN** Git仓库内部的子目录
- **THEN** 系统不将其标记为Git仓库根目录
- **AND** 显示为普通文件夹

### Requirement: 隐藏文件过滤
系统应过滤.git隐藏文件夹，不在文件树中显示。

#### Scenario: 过滤.git文件夹
- **WHEN** 系统扫描目录内容
- **THEN** 系统不包含.git文件夹在返回的节点列表中
- **AND** 用户无法在文件树中看到.git文件夹

#### Scenario: 显示其他隐藏文件
- **WHEN** 目录包含其他以.开头的文件或文件夹
- **THEN** 系统正常显示这些文件（除非未来添加更多过滤规则）

### Requirement: 文件树节点信息
系统应返回完整的文件树节点信息。

#### Scenario: 文件夹节点信息
- **WHEN** 系统返回一个文件夹节点
- **THEN** 节点包含：唯一ID、文件名、完整路径、类型（DIRECTORY）、是否Git仓库、是否有子节点

#### Scenario: 文件节点信息
- **WHEN** 系统返回一个文件节点
- **THEN** 节点包含：唯一ID、文件名、完整路径、类型（FILE）、是否有子节点（false）

#### Scenario: 节点ID唯一性
- **WHEN** 系统生成节点ID
- **THEN** ID使用文件的完整路径
- **AND** ID在同一文件树中唯一

### Requirement: 文件树全部展开
系统应支持一键展开文件树的所有节点。

#### Scenario: 全部展开功能
- **WHEN** 用户点击"全部展开"按钮
- **THEN** 系统递归加载所有子节点
- **AND** 系统展开所有折叠的文件夹
- **AND** 文件树显示完整的目录结构

#### Scenario: 全部展开大目录
- **WHEN** 文件树包含大量节点（如深层嵌套或多个项目）
- **THEN** 系统显示加载提示
- **AND** 系统分批加载节点，避免界面卡顿

### Requirement: 文件树全部收起
系统应支持一键收起文件树的所有展开节点。

#### Scenario: 全部收起功能
- **WHEN** 用户点击"全部收起"按钮
- **THEN** 系统收起所有展开的文件夹
- **AND** 只保留根目录显示
- **AND** 不重新加载节点，仅改变UI显示状态

#### Scenario: 收起后重新展开
- **WHEN** 用户在全部收起后点击根目录的展开按钮
- **THEN** 系统显示之前加载的子节点（从内存）
- **AND** 不需要重新请求后端接口

### Requirement: 文件树节点选择
系统应支持用户选择文件树中的节点。

#### Scenario: 选择文件夹
- **WHEN** 用户点击文件夹名称
- **THEN** 系统高亮显示该文件夹
- **AND** 系统在右侧面板显示该文件夹的信息
- **AND** 系统提供文件夹操作按钮

#### Scenario: 选择文件
- **WHEN** 用户点击文件名称
- **THEN** 系统高亮显示该文件
- **AND** 系统在右侧面板显示该文件的信息
- **AND** 系统提供文件操作按钮

#### Scenario: 切换选择
- **WHEN** 用户点击另一个节点
- **THEN** 系统取消之前节点的选中状态
- **AND** 系统高亮显示新选中的节点
- **AND** 系统更新右侧面板内容

### Requirement: 文件树节点重载
系统应支持在文件操作后刷新文件树。

#### Scenario: 创建文件后刷新
- **WHEN** 用户在当前目录创建新文件或文件夹
- **THEN** 系统自动刷新当前文件夹的子节点列表
- **AND** 新创建的项出现在文件树中

#### Scenario: 删除文件后刷新
- **WHEN** 用户删除文件或文件夹
- **THEN** 系统自动刷新父文件夹的子节点列表
- **AND** 被删除的项从文件树中移除

#### Scenario: 重命名后刷新
- **WHEN** 用户重命名文件或文件夹
- **THEN** 系统自动刷新父文件夹的子节点列表
- **AND** 文件树显示新的名称

### Requirement: 文件类型图标
系统应根据文件类型显示不同的图标。

#### Scenario: 文件夹图标
- **WHEN** 节点类型为DIRECTORY
- **THEN** 系统显示文件夹图标

#### Scenario: Git仓库图标
- **WHEN** 文件夹是Git仓库根目录
- **THEN** 系统显示特殊的Git仓库标识（如 ✓）

#### Scenario: 文件图标
- **WHEN** 节点类型为FILE
- **THEN** 系统显示文件图标
