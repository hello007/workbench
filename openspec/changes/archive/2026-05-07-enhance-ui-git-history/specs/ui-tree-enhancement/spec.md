## ADDED Requirements

### Requirement: 文件树显示节点图标
系统 SHALL 在文件树中为每个节点显示适当的图标，以便用户直观区分文件类型和状态。

#### Scenario: 文件夹节点显示文件夹图标
- **WHEN** 用户查看文件树中的文件夹节点
- **THEN** 系统应显示文件夹图标（Folder 或 FolderOpened）
- **AND** 未展开的文件夹显示灰色文件夹图标
- **AND** 已展开的文件夹显示蓝色文件夹图标

#### Scenario: 文件节点显示文件图标
- **WHEN** 用户查看文件树中的文件节点
- **THEN** 系统应显示文件图标（Document）
- **AND** 图标颜色为深灰色（#606266）

#### Scenario: Git 仓库节点显示特殊标识
- **WHEN** 用户查看文件树中的 Git 仓库根目录节点
- **THEN** 系统应在节点右侧显示绿色的 Git 图标或徽章
- **AND** 标识清晰可见，不与节点文本重叠

### Requirement: 文件树支持颜色主题
系统 SHALL 为文件树节点应用统一的颜色主题，以提升视觉层次感。

#### Scenario: 文件夹节点颜色主题
- **WHEN** 用户查看文件树中的文件夹节点
- **THEN** 未展开的文件夹文本颜色为 #909399（灰色）
- **AND** 已展开的文件夹文本颜色为 #409EFF（蓝色）

#### Scenario: 文件节点颜色主题
- **WHEN** 用户查看文件树中的文件节点
- **THEN** 文件节点文本颜色为 #606266（深灰色）

#### Scenario: Git 仓库标识颜色
- **WHEN** 用户查看文件树中的 Git 仓库节点
- **THEN** Git 标识图标颜色为 #67C23A（绿色）

### Requirement: 文件树支持悬停效果
系统 SHALL 在用户悬停在文件树节点上时提供视觉反馈，以改善交互体验。

#### Scenario: 悬停在节点上显示背景色
- **WHEN** 用户将鼠标悬停在文件树节点上
- **THEN** 节点背景色应变为 #F5F7FA（浅灰色）
- **AND** 背景色变化应使用平滑的 CSS transition（持续时间 200ms）

#### Scenario: 悬停效果不影响节点功能
- **WHEN** 用户悬停在节点上时点击节点
- **THEN** 系统应正常触发节点点击事件
- **AND** 悬停效果不应阻止节点的展开/折叠操作

### Requirement: 文件树支持展开/折叠动画
系统 SHALL 在节点展开或折叠时提供平滑的动画效果，提升视觉体验。

#### Scenario: 展开节点时显示动画
- **WHEN** 用户点击展开文件夹节点
- **THEN** 子节点应以平滑动画展开（持续时间 300ms）
- **AND** 动画应使用硬件加速（CSS transform）

#### Scenario: 折叠节点时显示动画
- **WHEN** 用户点击折叠文件夹节点
- **THEN** 子节点应以平滑动画折叠（持续时间 300ms）
- **AND** 动画应使用硬件加速（CSS transform）

#### Scenario: 动画不影响性能
- **WHEN** 用户快速展开/折叠多个节点
- **THEN** 动画应保持流畅，不低于 30fps
- **AND** 不应导致 UI 卡顿或响应延迟

### Requirement: 文件树图标自适应节点状态
系统 SHALL 根据节点的展开状态动态切换文件夹图标。

#### Scenario: 展开文件夹后图标变化
- **WHEN** 用户展开文件夹节点
- **THEN** 系统应将文件夹图标从 Folder 切换为 FolderOpened
- **AND** 图标切换应立即生效，无延迟

#### Scenario: 折叠文件夹后图标变化
- **WHEN** 用户折叠文件夹节点
- **THEN** 系统应将文件夹图标从 FolderOpened 切换为 Folder
- **AND** 图标切换应立即生效，无延迟

### Requirement: 文件树支持自定义节点样式
系统 SHALL 允许为不同类型的节点应用自定义样式，以支持未来的扩展需求。

#### Scenario: Git 仓库节点样式定制
- **WHEN** 系统渲染 Git 仓库根目录节点
- **THEN** 系统应为节点应用特殊的 CSS 类名（如 `.is-git-repo`）
- **AND** 该类名可用于额外的样式定制或行为绑定

#### Scenario: 特殊文件类型样式定制
- **WHEN** 系统识别特殊文件类型（如 .md、.json、.gitignore）
- **THEN** 系统应能够为这些文件类型应用不同的图标或颜色
- **AND** 样式定制应基于文件扩展名或文件名模式
