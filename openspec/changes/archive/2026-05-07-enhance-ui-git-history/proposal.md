## Why

当前的 Git 管理工具在用户体验和 Git 功能展示方面存在不足：左侧文件树样式较为简陋，缺乏视觉层次感；对于 Git 仓库的信息展示不够完整，未显示远程仓库地址等关键信息；缺少 Git 历史提交查看功能，限制了用户对代码仓库历史的追溯和管理能力。这些问题影响了工具的实用性和专业性。

## What Changes

- **左侧树样式美化**：优化文件树的视觉效果，包括节点图标、颜色主题、展开/折叠动画、悬停效果等，提升视觉层次感和交互体验
- **Git 仓库地址信息展示**：在右侧区域增加 Git 仓库的详细信息展示，包括远程仓库地址（origin URL）、当前分支、提交信息等关键元数据
- **Git 历史提交查看功能**：增加提交历史浏览功能，支持查看提交列表、提交详情（作者、时间、消息、变更文件），并提供提交间的快速导航

## Capabilities

### New Capabilities
- `ui-tree-enhancement`: 增强的 UI 树组件样式和交互体验
- `git-info-display`: Git 仓库详细信息的展示功能
- `git-commit-history`: Git 历史提交的查看和浏览功能

### Modified Capabilities
*无*

## Impact

- **前端组件**：需要修改 `Home.vue` 中的文件树组件样式，新增 Git 信息展示组件和提交历史组件
- **Go 后端**：可能需要新增或扩展 Go 方法以支持获取 Git 仓库地址信息和提交历史数据
- **API 接口**：如果需要新增 Go 方法，将在 `wailsjs` 绑定中自动生成相应的 JavaScript 接口
- **第三方依赖**：可能需要引入 Element Plus 的更多组件（如 Timeline、Card、Tag 等）以支持新的 UI 需求
- **数据模型**：可能需要扩展数据结构以承载更丰富的 Git 仓库和提交历史信息
