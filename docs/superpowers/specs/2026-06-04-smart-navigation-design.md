# 智能导航中心设计文档

## 1. 概述

**功能名称：** 智能导航中心（Smart Navigation Center）
**目标：** 将收藏夹、快速搜索、最近访问、工作目录切换统一为 Command Palette 式交互入口，配合树状态记忆和深层目录支持，大幅提升文件导航效率。

**核心痛点：**
1. 切换工作目录时展开状态丢失，需要重新展开
2. 工作目录少时文件树层级太深，定位目标文件慢

## 2. 树状态记忆（基础设施层）

### 2.1 机制

- 每个工作目录维护一份独立的展开节点列表，key 为工作目录路径
- 存储位置：localStorage，key 格式 `treeState:{dirPath}`
- 存储内容：`{ expandedPaths: string[], scrollTop: number, selectedPath: string | null }`
- 切换工作目录时自动保存当前状态，加载目标目录的已保存状态
- 如果目标目录无保存状态，保持默认收起行为

### 2.2 容量控制

- 每个目录最多记忆 200 个展开节点，超出时按最早展开时间淘汰

### 2.3 生命周期

- 删除工作目录时清除对应的状态数据
- 应用关闭/刷新时状态持久化（localStorage 天然支持）

## 3. 智能导航中心（Command Palette）

### 3.1 触发方式

- 快捷键 `Ctrl+P` 打开
- ActivityBar 增加搜索图标按钮
- 文件树顶部工具栏增加搜索按钮

### 3.2 UI 形态

- 居中弹出的模态输入框（类似 VSCode Command Palette）
- 顶部为搜索输入框，下方为结果列表
- 支持键盘上下选择、Enter 确认、Esc 关闭

### 3.3 搜索模式

| 前缀 | 模式 | 说明 |
|------|------|------|
| 无前缀 | 综合搜索 | 搜索文件名、目录名、收藏项 |
| `>` | 命令 | 预留，未来可扩展为操作命令 |
| `@` | 收藏夹 | 只搜索收藏的路径 |
| `#` | 工作目录切换 | 切换当前工作目录 |

### 3.4 搜索范围

- 默认搜索当前工作目录下的文件和目录
- 收藏夹条目始终参与搜索（不受当前工作目录限制）
- 模糊匹配，支持路径片段（如输入 `svc/user` 匹配 `service/user-center`）

### 3.5 结果列表

- 分组展示：收藏夹命中 → 最近访问 → 文件搜索结果
- 每条显示：图标 + 文件/目录名 + 相对路径（灰色）
- 最多显示 20 条结果，输入变化时实时过滤

### 3.6 选中后的行为

- 文件：在文件树中定位并选中，触发预览
- 目录：在文件树中展开并定位到该目录
- 工作目录：切换到对应工作目录（触发树状态恢复）

## 4. 收藏夹系统

### 4.1 添加收藏的方式

- 文件树右键菜单增加"添加到收藏"选项（文件和目录均支持）
- Command Palette 搜索结果中，`Ctrl+D` 快捷收藏选中项
- 收藏夹管理面板中手动添加路径

### 4.2 数据结构

```json
{
  "path": "string",
  "alias": "string | null",
  "group": "string",
  "createdAt": "number"
}
```

### 4.3 存储

- 存入应用配置文件（与工作目录配置同级），跟随 data/ 目录持久化
- 不绑定特定工作目录，全局可用

### 4.4 收藏夹管理

- 在 Command Palette 中输入 `@` 前缀查看和搜索所有收藏
- 支持分组管理（拖拽调整分组、重命名分组）
- 右键菜单：重命名别名、移动分组、移除收藏
- 收藏上限：100 条

### 4.5 跳转行为

- 如果收藏路径在当前工作目录下：直接在文件树中定位展开
- 如果收藏路径在其他工作目录下：自动切换工作目录，再定位
- 如果收藏路径不属于任何已添加的工作目录：提示用户是否将其父目录添加为工作目录

## 5. 深层目录作为工作目录

### 5.1 实现方式

- 添加工作目录时不限制层级，支持任意深度的子目录
- 文件树右键菜单增加"添加为工作目录"选项（仅对目录节点显示）
- 工作目录列表中显示完整路径，避免同名目录混淆

### 5.2 UI 调整

- 工作目录列表中，如果路径过长，显示为：`...父目录/当前目录名`，鼠标悬停显示完整路径
- 工作目录卡片增加路径层级视觉提示

### 5.3 防重复逻辑

- 添加工作目录前检查是否已存在父子关系的目录
- 如果新添加的目录是已有工作目录的父目录，提示"已存在子目录 X，是否替换？"
- 如果新添加的目录是已有工作目录的子目录，提示"已存在父目录 Y，是否继续添加？"（允许共存）

### 5.4 Git 检测

- 深层工作目录依然支持 Git 仓库检测（向上查找 .git 目录）
- 右键菜单的"更新仓库"操作基于检测到的 .git 根目录执行

## 6. 最近访问历史

### 6.1 记录范围

- 记录用户点击选中的文件和目录（不含鼠标悬停）
- 记录通过 Command Palette 跳转的路径
- 记录通过收藏夹跳转的路径

### 6.2 存储

- localStorage，key 为 `recentAccess`
- 存储结构：

```json
{
  "path": "string",
  "type": "file | dir",
  "workDir": "string",
  "lastAccess": "number"
}
```

### 6.3 展示位置

- Command Palette 中，无输入时默认显示最近 10 条访问记录
- 显示格式：图标 + 文件/目录名 + 相对路径 + 访问时间（如"2分钟前"）

### 6.4 清理策略

- 最多保留 50 条记录
- 超出时淘汰最老的记录
- 如果访问的路径已被删除，下次启动时自动清理

### 6.5 隐私选项

- 设置面板中增加"清除最近访问记录"按钮
- 增加"启用/禁用最近访问记录"开关

## 7. 交互流程与边界处理

### 7.1 典型使用流程

1. **快速跳转文件：** `Ctrl+P` → 输入文件名片段 → `Enter` → 文件树定位并预览
2. **切换工作目录：** `Ctrl+P` → 输入 `#` → 选择目标工作目录 → 树状态自动恢复
3. **使用收藏夹：** `Ctrl+P` → 输入 `@` → 选择收藏项 → 自动定位（必要时切换工作目录）
4. **添加深层目录：** 文件树右键目录 → "添加为工作目录" → 直接加入工作目录列表

### 7.2 性能考虑

- 搜索采用防抖（300ms），避免每次输入都触发全量搜索
- 文件搜索结果缓存（按输入前缀缓存），输入变化时增量过滤
- 大型目录（超过 5000 文件）搜索时显示加载指示器
- 树状态恢复采用异步批量展开，避免阻塞 UI

### 7.3 错误处理

- 收藏路径不存在时，在 Command Palette 中标记为"已失效"，允许用户移除
- 工作目录被删除后，下次启动自动从配置中移除
- 树状态恢复失败（路径不存在）时静默跳过，不阻断加载
- localStorage 存储失败时降级为会话内存存储，关闭应用后丢失

### 7.4 冲突处理

- 如果同时打开多个 Git Manager 窗口，localStorage 更新采用"最后写入优胜"策略
- 树状态和收藏夹配置支持实时同步（监听 storage 事件）

### 7.5 快捷键冲突

- `Ctrl+P` 如果与系统或其他功能冲突，允许在设置中重新绑定
- 预留 `Ctrl+Shift+P` 作为备用快捷键
