# 工具箱功能设计

## 概述

在 WorkBench 主界面增加"工具箱"功能入口，将全局常用工具（与具体文件无关的指令）集中管理。首个工具为"拷贝到"，后续可扩展。

## 设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 入口形式 | 左侧活动栏 + 浮动覆盖面板 | 不占用额外布局空间，与现有三栏布局兼容 |
| 面板与目录树关系 | 浮动覆盖目录树区域 | 不改变现有 Splitpanes 布局，工具箱作为覆盖层 |
| 工具操作方式 | 点击工具项 → 弹出对话框 | 与现有"拷贝到"交互一致 |
| 现有入口 | 保留右键菜单和 ContentPanel 中的"拷贝到" | 多入口共存，不破坏已有使用习惯 |

## 布局结构

```

│      │               │            │              │
│ 📁   │  项目A        │  src/      │              │
│ 🧰   │  项目B ✓      │  main.go   │              │

│      │               │            │              │

```

```
┌──────┬──────────────┬────────────┬──────────────┐
│活动栏 │🧰 工具箱  ✕  │  文件树     │  内容面板     │

        ↑ 浮动覆盖目录树区域
```

## 交互流程

1. 点击活动栏 🧰 图标 → 工具箱面板浮动覆盖目录树区域，🧰 图标高亮
2. 点击工具项（如"拷贝到"）→ 弹出"拷贝到"对话框，与现有逻辑一致
3. 点击活动栏 📁 图标 / 面板右上角 ✕ → 关闭工具箱，恢复目录树显示

## 组件设计

### ActivityBar 组件（新增）

- 位置：窗口最左侧，宽度 44px
- 内容：垂直排列的图标按钮
- 图标列表：
  - 📁 工作目录（默认选中，对应 DirectoryTree）
  - 🧰 工具箱（对应 ToolboxPanel）
- 选中状态：高亮背景色 + 外发光

### ToolboxPanel 组件（新增）

- 位置：浮动覆盖 DirectoryTree 区域
- 宽度：继承 DirectoryTree 的 Pane 宽度
- 结构：
  - 标题栏：🧰 工具箱 + ✕ 关闭按钮
  - 工具列表：卡片式工具项
- 工具项结构：图标 + 名称 + 描述

### 工具项定义

```js
{
  id: 'copyTo',
  name: '拷贝到',
  icon: 'CopyDocument',  // Element Plus 图标
  description: '将文件/文件夹拷贝到指定目录',
  handler: 'showCopyToDialog'  // 触发动作
}
```

- 无变更

### FileTreePanel.vue

- 无变更（保留按钮组中的"拷贝到"入口）

## 扩展性

新增工具只需：
1. 在工具项配置中添加一项
2. 实现对应的 handler 函数
3. 如需对话框，在 ToolboxPanel 中新增对话框组件

后续可能扩展的工具示例：
- 批量重命名
- 文件搜索
- 仓库健康检查
