# 拷贝到对话框 — 源/目标路径互换功能设计

> **日期：** 2026-06-09
> **状态：** 已确认，待实施
> **涉及文件：** `FileTreePanel.vue`、`ToolboxPanel.vue`

---

## 1. 背景

当前"拷贝到"对话框包含"原地址"和"目标地址"两个输入框，用户需要手动编辑才能互换路径。在需要反向操作（如从目标地址拷贝回源地址）时不够便捷。

## 2. 需求

在"原地址"与"目标地址"两个输入框之间添加一个**互换按钮**，点击后交换两个字段的值。

## 3. 设计

### 3.1 UI 布局

在两个 `el-form-item` 之间插入一行居中的互换按钮：

```
┌──────────────────────────┐
│ 原地址   [ /src/path   ] │
│        [ ⇅ 互换 ]        │
│ 目标地址 [ /dst/path   ] │
│ □ 包含文件夹本身          │
└──────────────────────────┘
```

### 3.2 按钮规格

| 属性     | 值                                     |
|----------|----------------------------------------|
| 图标     | Element Plus `Sort` 图标               |
| 文字     | "互换"                                 |
| 样式     | `text` 类型按钮，居中对齐，小尺寸       |
| 交互反馈 | 点击时图标短暂旋转动画（CSS transition）|

### 3.3 行为逻辑

1. 点击互换按钮 → 交换 `copyToSourcePath` 和 `copyToTargetPath` 的值
2. 拷贝预览区自动更新（已是 computed 属性）
3. 不影响"包含文件夹本身"复选框状态
4. 按钮在拷贝进行中（`copyToLoading = true`）时禁用

### 3.4 影响范围

| 组件                  | 改动   |
|-----------------------|--------|
| `FileTreePanel.vue`   | 添加互换按钮（模板 + 样式） |
| `ToolboxPanel.vue`    | 同上，保持体验一致           |
| 后端                  | 无改动                       |

## 4. 不涉及

- 后端逻辑变更
- "包含文件夹本身"复选框状态联动
- 现有校验流程变更
