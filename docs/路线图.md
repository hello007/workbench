# WorkBench 发展路线图

本文档规划了 WorkBench 项目的未来发展方向和功能计划。

## Phase 2 功能（计划中）

### Git 增强功能

- [ ] **Git 提交和推送**
  - 支持选择文件进行暂存
  - 编辑提交信息
  - 推送到远程仓库
  - 查看推送结果

- [ ] **分支管理**
  - 查看所有分支（本地和远程）
  - 创建新分支
  - 切换分支
  - 删除分支
  - 重命名分支

- [ ] **暂存区和 Diff 查看**
  - 查看未暂存的更改
  - 查看已暂存的更改
  - 行级 diff 对比
  - 暂存/取消暂存单个文件
  - 丢弃本地更改

- [ ] **提交历史增强**
  - 查看提交详情（作者、日期、文件列表）
  - 查看提交之间的 diff
  - 搜索提交历史
  - 按作者/日期/文件过滤

### 用户界面增强

- [x] **右键菜单实现**
  - 文件夹：新建文件夹、新建文件、重命名、删除
  - 文件：重命名、删除、打开方式
  - 剪切/复制/粘贴/拷贝到
  - 在资源管理器/VSCode/Warp 中打开
  - 用默认程序打开文件

- [x] **文件搜索功能**
  - 按文件名搜索 ✅
  - 按文件内容搜索（基于 ripgrep） ✅
  - 搜索排除规则配置 ✅
  - 快捷键支持（Ctrl+P） ✅

- [x] **多仓库批量操作**
  - 批量拉取更新
  - 实时进度展示

### 用户体验优化

- [ ] **暗色主题支持**
  - 系统主题自动跟随
  - 手动切换亮色/暗色主题
  - 主题配置持久化

- [x] **快捷键支持**
  - 常用操作的快捷键
  - 快捷键配置面板
  - 快捷键提示（右键菜单）

- [ ] **多语言支持**
  - 英文界面
  - 中文界面（已有）
  - 语言切换功能
  - i18n 国际化框架

## 性能优化

### 文件树优化

- [ ] **大型仓库文件树加载优化**
  - 虚拟滚动技术
  - 分批异步加载
  - 加载进度指示器
  - 可配置的加载深度限制

- [ ] **文件树缓存**
  - 内存缓存已加载的节点
  - 持久化缓存到本地文件
  - 智能缓存失效策略
  - 缓存大小限制

### Git 操作优化

- [ ] **提交历史缓存**
  - 本地缓存提交历史
  - 增量更新机制
  - 缓存过期策略

- [ ] **并发操作控制**
  - 限制同时进行的 Git 操作数量
  - 操作队列管理
  - 操作取消功能
  - 进度显示优化

### 应用性能

- [ ] **启动时间优化**
  - 延迟加载非关键模块
  - 优化配置文件读取
  - 减少 HTTP 请求

- [ ] **内存使用优化**
  - 及时释放不再使用的对象
  - 限制文件树节点数量
  - 大文件分块读取
