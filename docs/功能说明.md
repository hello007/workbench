# WorkBench 功能说明

## 1. 工作目录管理

- 添加/删除/更新工作目录
- 设置默认目录
- 拖拽排序目录顺序
- JSON持久化配置
- 路径验证（Windows格式）
- 右键菜单：重命名（F2）、设为默认、复制路径、在资源管理器中打开、用 VSCode 打开、用 Warp 打开、用 Obsidian 打开（以该目录作为 Obsidian 仓库）、更新仓库、删除（Delete）
- Git 仓库标记：本身是 git 仓库的工作目录项显示 git 图标；已配置远程显示彩色 git.png，未配置远程显示灰色 git-gray.png，便于一眼识别

## 2. 文件树浏览

- 懒加载机制（按需加载子节点）
- Git仓库自动检测（git 图标标识，彩色/灰色区分有无远程）
- .git文件夹过滤
- 全部展开/收起功能
- 节点选择和高亮
- 点击祖先目录时仅选中不收起（向上导航保护）

## 3. 文件操作

- 创建文件/文件夹
- 重命名、删除
- 文件预览（支持文本/二进制检测）
- unsupported 按文本预览 - 无扩展名或未知扩展名文件优先尝试按文本读取（UTF-8 优先 + GBK 兜底解码），可显示则降级为文本预览；含 NUL 字节或解码失败的二进制文件、超大文件（>1MB）走降级提示
- 文件预览编辑 — 预览文本文件时可就地编辑，修改后保存或取消，切换文件时自动检查未保存修改；编辑态支持 Ctrl+S 保存、Esc 取消（有未保存修改时二次确认）；保存按原文件编码（UTF-8/GBK）写入，不改变原文件编码（GBK 文件编辑后仍为 GBK）
- Markdown 预览链接导航 — 相对引用（./other.md、../readme.md）在预览面板内切换预览；外部 http 链接用系统浏览器打开；同文档锚点（#标题）滚动定位
- Markdown 预览增强 — 支持 mermaid 图形渲染（流程图/时序图等，渲染失败降级为代码文本）；右侧可折叠标题目录（TOC），点击快速跳转到对应标题
- Markdown frontmatter 属性面板 — 文档开头的 YAML frontmatter（`---\n...\n---`）不再被当作普通正文渲染，而是解析为正文上方的结构化属性表格：数组值显示为标签徽章，标量值原样展示，嵌套对象以 JSON 文本呈现；解析失败时降级为带语法高亮的 YAML 原文代码块，不影响正文
- 文件大小限制（1MB）
- 剪切/复制/粘贴操作
- 拷贝到对话框（支持预览效果）
- 用默认程序打开文件
- 右键菜单操作（新建文件/文件夹、重命名、删除、剪切/复制/粘贴、拷贝到、复制路径/文件名、在资源管理器中打开、用 VSCode/Warp 打开、用 Obsidian 打开、更新仓库）
- 用 Obsidian 打开：文件夹→以自身作为仓库，文件→以父目录作为仓库
- 打开前预检：读取 Obsidian 仓库注册表（%APPDATA%\obsidian\obsidian.json）判断目录是否属于已注册 vault；未注册时弹三按钮确认框提供两条路径，避免 Vault not found 错误；注册表读取失败则降级为现状尽力打开
  - 「自动注册并打开」：二次确认（预告首次打开信任提示、配置已备份、运行中需关闭）后，检测 Obsidian 进程未运行则自动向 obsidian.json 追加 vault 条目（备份+原子写，保留未知顶层字段）并发 URI 打开；运行中则提示关闭后重试（不强制杀进程）。**注意：Obsidian 运行时持有 vault 列表内存缓存，运行中写入会被其退出/切换仓库时回写覆盖导致注册丢失，故自动注册必须在 Obsidian 未运行时进行；首次打开新注册仓库仍会弹「Do you trust the author」信任提示，需用户手动确认。**
  - 「打开仓库管理器」：复制目录路径到剪贴板并跳转 obsidian://choose-vault 手动添加
- 按节点类型区分菜单项（文件夹/文件显示不同操作）

## 4. Git集成

- 查看仓库信息（分支、远程、提交历史）
- 克隆远程仓库（HTTPS/SSH）
- 拉取更新
- 切换分支
- 查看本地变动列表（单区变动清单，IDEA commit 窗口风格）
- 文件差异比较（diff）：双击变动文件，弹窗以左右双栏对照展示差异（已跟踪文件对比 HEAD，未跟踪文件展示为新增全文；二进制、图片、超大文件给出友好提示）
- 选择性提交（commit）：通过复选框勾选本次要提交的文件，输入提交信息后点击【提交】，仅提交勾选文件（pathspec 语义，未勾选文件保持未提交）
- 提交并推送、推送到远程（push）：当前分支无上游时弹确认是否 `git push --set-upstream origin <分支>`，凭证透传系统 git 配置
- 回滚变动：保留对勾选文件的回滚入口（含二次确认），下沉到「更多」下拉菜单
- 提交/推送后联动刷新本地变动列表、提交历史与仓库信息
- 丢弃本地更改
- 分页查看提交历史（默认10条/页）
- 批量更新仓库（递归扫描子目录，并行拉取，实时进度展示；未配置远程的本地仓库自动跳过，不计失败）
- 工作目录直显仓库详情：在左栏选中一个 git 仓库工作目录时，右侧操作面板直接显示该仓库的仓库信息、提交历史、本地变动（与在文件树中选中 git 仓库节点一致），无需再到文件树点选；非 git 工作目录选中后保持原空状态

## 5. 智能导航中心

- Command Palette 快速导航（Ctrl+P）
- 文件/目录模糊搜索
- 内容搜索（基于 ripgrep，支持文件扩展名过滤、排除规则配置、指定目录搜索）
- 收藏夹系统（添加、分组、跳转、右键移除）
- 工作目录快速切换（#前缀）
- 每个工作目录独立记忆展开状态
- 最近访问历史
- 深层子目录可作为工作目录
- 长路径省略显示（鼠标悬停显示完整路径）

## 6. 内置终端

- 底部面板集成完整终端，支持 PowerShell / CMD / Git Bash / WSL 切换
- 工作目录自动跟随文件树
- 可拖拽调节面板高度
- Ctrl+` 快速切换（快捷键可自定义）
- 首次打开即显示 Shell 提示符
- 设置面板可配置默认 Shell 类型和自定义路径

## 7. 自定义快捷键

- 设置面板「快捷键」标签页管理
- 支持「打开命令面板（Ctrl+P）」「切换终端（Ctrl+`）」「重命名（F2）」「删除（Delete）」四个快捷键
- 点击录制新快捷键，实时显示组合键（支持 Ctrl/Alt/Shift 组合键，以及 F1-F12、Delete 等功能键单键；字母/数字单键不允许，避免与文本输入冲突）
- 重命名/删除快捷键作用于"最近交互的树面板"（工作目录列表或文件树）的当前选中项；输入框/对话框/终端聚焦时不触发，避免误触（文件树删除为永久删除）
- 右键菜单显示对应快捷键提示
- 单个重置 / 全部重置为默认值
- 配置持久化到 settings.json

## 8. 设置面板

- 弹窗式设置面板，左右双栏布局
- 通用设置、终端设置、搜索设置、快捷键设置四个标签页
- 通用→外部应用：可配置 Obsidian 程序路径，配置后"用 Obsidian 打开"优先使用该可执行文件；留空则尝试系统已注册的 Obsidian
- 设置项自动保存持久化
