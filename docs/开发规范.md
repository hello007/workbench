# Git Manager 开发规范

本文档定义了 Git Manager 项目的开发规范和最佳实践。

## 代码风格

### Go 代码规范

**基本原则：**
- 遵循标准 Go 代码规范
- 使用 `gofmt` 自动格式化
- 导出函数使用 PascalCase
- 私有函数使用 camelCase

**示例：**
```go
// ✅ 正确：导出函数使用 PascalCase
func GetDirectories() []*model.Directory {
    // ...
}

// ✅ 正确：私有函数使用 camelCase
func validatePath(path string) error {
    // ...
}

// ❌ 错误：导出函数使用 camelCase
func getDirectories() []*model.Directory {
    // ...
}
```

**命名约定：**
- 文件名：小写，使用下划线分隔（如 `file_tree.go`）
- 接口名：以 `-er` 结尾（如 `Reader`, `Writer`）
- 常量：全大写，使用下划线分隔（如 `MAX_FILE_SIZE`）

### Vue 代码规范

**使用 Composition API：**
```vue
<script setup>
// ✅ 推荐：使用 <script setup>
import { ref, onMounted } from 'vue'

const count = ref(0)
onMounted(() => {
  console.log('mounted')
})
</script>
```

**组件命名：**
- 多词组件名（避免与 HTML 标签冲突）
- PascalCase 命名（如 `UserProfile.vue`, `NavBar.vue`）

**属性命名：**
- camelCase 在 JavaScript 中
- kebab-case 在模板中

```vue
<!-- ✅ 正确 -->
<UserCard user-name="John" />

<!-- ❌ 错误 -->
<UserCard userName="John" />
```

### 注释规范

**基本原则：仅在 WHY 非显而易见时添加注释**

```go
// ✅ 好的注释：解释为什么
// 使用缓存避免重复计算斐波那契数列
func fibonacci(n int) int {
    // ...
}

// ❌ 不好的注释：重复代码内容
// 计算斐波那契数列
func fibonacci(n int) int {
    // ...
}
```

**注释长度：**
- 单行注释最多一行
- 不使用多行注释块
- 如果需要解释，重构代码使其自解释

## 调试日志

### 前端调试日志

**使用 `utils/debug.js` 工具：**

```javascript
import { debug } from '../utils/debug'

// ✅ 正确：使用 debug 工具（开发环境输出，生产环境静默）
debug.log('Loading data:', data)
debug.error('Error occurred:', error)

// ❌ 错误：直接使用 console.log（生产环境会泄露）
console.log('Loading data:', data)
```

**调试工具特性：**
- `debug.log()` - 仅在开发环境输出
- `debug.error()` - 始终输出（错误日志）
- `debug.warn()` - 仅在开发环境输出警告

### 后端调试日志

**使用 `println()` 输出到终端：**

```go
// ✅ 正确：使用 println 输出调试信息
println("Loading directories...")
println("Error:", err.Error())

// ❌ 错误：使用 log 包（不推荐在 Wails 中使用）
log.Printf("Loading directories...")
```

**日志级别：**
- `println()` - 一般信息（仅在开发时）
- 返回 error - 错误信息（始终返回）

## 错误处理

### Go 错误处理

**基本原则：显式检查 error，返回有意义的错误信息**

```go
// ✅ 正确：显式检查错误
func LoadDirectories() ([]*Directory, error) {
    data, err := os.ReadFile(configPath)
    if err != nil {
        return nil, fmt.Errorf("无法读取配置文件: %w", err)
    }
    // ...
}

// ❌ 错误：忽略错误
func LoadDirectories() []*Directory {
    data, _ := os.ReadFile(configPath) // 忽略错误
    // ...
}
```

**错误包装：**
```go
// 使用 fmt.Errorf 包装错误，保留原始错误信息
if err != nil {
    return nil, fmt.Errorf("加载目录失败: %w", err)
}
```

### 前端错误处理

**使用 ElMessage 显示用户友好的错误提示：**

```javascript
// ✅ 正确：使用 ElMessage 显示错误
try {
  const nodes = await GetFileTree(path)
  // ...
} catch (error) {
  ElMessage.error('加载节点失败: ' + (error.message || error))
}

// ❌ 错误：直接 console.error
try {
  // ...
} catch (error) {
  console.error(error) // 用户看不到
}
```

**运算符优先级注意事项：**

```javascript
// ❌ 错误：运算符优先级导致错误
ElMessage.error('加载失败: ' + error.message || error)
// 等价于：('加载失败: ' + error.message) || error

// ✅ 正确：使用括号明确意图
ElMessage.error('加载失败: ' + (error.message || error))
```

## Git 提交规范

### 提交消息格式

```
<type>: <subject>
```

**类型（type）：**
- `feat` - 新功能
- `fix` - 修复 bug
- `refactor` - 重构代码
- `docs` - 文档更新
- `test` - 测试相关
- `chore` - 构建/工具链相关

**示例：**
```bash
feat: 添加文件预览功能
fix: 修复文件树懒加载根节点检测逻辑
refactor: 重构目录服务，分离配置和业务逻辑
docs: 更新开发文档，添加常见问题章节
test: 添加文件树组件单元测试
```

### 提交消息最佳实践

**使用中文（与项目语言一致）：**
```bash
# ✅ 正确
feat: 添加工作目录管理功能

# ❌ 避免（除非团队约定使用英文）
feat: Add workspace directory management feature
```

**使用祈使句：**
```bash
# ✅ 正确
fix: 修复文件预览不更新的问题

# ❌ 错误
fix: 修复了文件预览不更新的问题
fix: 文件预览不更新问题修复
```

**说明为什么（而不是是什么）：**
```bash
# ✅ 正确
fix: 修复懒加载根节点检测以支持 Element Plus Tree 的 node.level 属性

# ❌ 错误
fix: 修改 if 条件判断
```

## 性能规范

### 懒加载

**文件树使用懒加载：**
```javascript
// ✅ 正确：按需加载子节点
const loadTreeNode = async (node, resolve) => {
  const nodes = await GetFileTree(node.data.path)
  resolve(nodes)
}

// ❌ 错误：一次性加载整个树
const loadAllNodes = async () => {
  const allNodes = await getAllFilesRecursive(rootPath)
  return allNodes
}
```

### 防抖和节流

**频繁操作使用防抖：**
```javascript
// ✅ 正确：手写防抖函数（项目不引入额外 npm 依赖）
function debounce(fn, delay) {
  let timer
  return (...args) => {
    clearTimeout(timer)
    timer = setTimeout(() => fn(...args), delay)
  }
}
```

## 安全规范

### 输入验证

**验证用户输入：**
```go
// ✅ 正确：验证路径
func validatePath(path string) error {
    if path == "" {
        return errors.New("路径不能为空")
    }
    if strings.ContainsAny(path, `<>:"|?*`) {
        return errors.New("路径包含非法字符")
    }
    return nil
}
```

### 权限检查

**文件操作前检查权限：**
```go
// ✅ 检查文件是否存在
if _, err := os.Stat(path); os.IsNotExist(err) {
    return errors.New("文件不存在")
}
```
