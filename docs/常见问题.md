# Git Manager 常见问题

本文档收集了 Git Manager 开发和使用过程中的常见问题及解决方案。

## 开发问题

### Q: 文件树不显示？

**症状：** 选择工作目录后，左侧文件树一片空白，控制台无错误。

**原因：** Element Plus Tree 的懒加载模式下，根节点检测逻辑不正确。

**解决方案：**

检查 `loadTreeNode` 函数中的根节点判断逻辑：

```javascript
// ❌ 错误的判断（永远为 false）
if (!node || Object.keys(node).length === 0) {
  // 加载根目录
}

// ✅ 正确的判断
if (!node || node.level === 0 || !node.data) {
  // 加载根目录
  const dir = directories.value.find(d => d.id === selectedDirectoryId.value)
  path = dir.path
}
```

**调试步骤：**
1. 打开浏览器控制台
2. 查看是否有 `loadTreeNode called` 日志
3. 检查 `node.level` 的值
4. 验证 `selectedDirectoryId` 是否正确

### Q: 切换节点时预览内容不清空？

**症状：** 点击不同文件/文件夹时，右侧预览区域仍显示上一个文件的内容。

**原因：** `onNodeClick` 函数没有重置 `filePreview` 状态。

**解决方案：**

在 `onNodeClick` 函数开始时清空预览状态：

```javascript
const onNodeClick = async (data) => {
  selectedNode.value = data

  // ✅ 添加：清空文件预览
  filePreview.value = {
    content: '',
    error: ''
  }

  // 如果是Git仓库，获取Git信息
  if (data.isGitRepo) {
    const info = await GetGitInfo(data.path)
    Object.assign(selectedNode.value, info)
  }
}
```

### Q: 端口被占用怎么办？

**症状：** 启动 `wails dev` 时提示端口 34115 被占用。

**解决方案：**

**方法 1：停止占用端口的进程**

```bash
# Windows: 查找占用端口的进程
netstat -ano | findstr ":34115"

# 输出示例：
# TCP    0.0.0.0:34115    0.0.0.0:0    LISTENING    12345
#                                              ↑
#                                           进程 PID

# 终止进程
taskkill /F /PID 12345

# 如果是 git-manager.exe，可以直接使用进程名
taskkill /F /IM git-manager.exe
```

**方法 2：使用其他端口**

```bash
wails dev -port 40000
```

**方法 3：重启电脑**

如果上述方法无效，重启电脑可以释放被占用的端口。

### Q: 前端依赖错误怎么办？

**症状：**
- `npm install` 失败
- `wails dev` 时前端无法启动
- 控制台提示模块找不到

**解决方案：**

**清理 node_modules 并重新安装：**

```bash
cd frontend

# 删除 node_modules 和 package-lock.json
rm -rf node_modules package-lock.json

# 重新安装依赖
npm install
```

**如果仍然失败，尝试：**

```bash
# 清理 npm 缓存
npm cache clean --force

# 删除 ~/.npm/_cacache
rm -rf ~/.npm/_cacache  # Linux/Mac
# 或手动删除 Windows 下的 npm 缓存目录

# 重新安装
npm install
```

**检查 Node.js 版本：**

```bash
node --version  # 应该是 16+ 或 18+
```

### Q: Wails 绑定生成失败？

**症状：**
- 前端调用 Go 方法时报错 `undefined`
- `wailsjs/go/main/App.js` 文件不存在或为空

**解决方案：**

**清理并重新生成绑定：**

```bash
cd frontend

# 删除 wailsjs 目录
rm -rf wailsjs

cd ..

# 重新生成绑定并启动
wails dev -clean
```

**检查 app.go 中的导出方法：**

```go
// ✅ 正确：导出方法（大写字母开头）
func (a *App) GetDirectories() []*model.Directory {
    // ...
}

// ❌ 错误：未导出方法（小写字母开头）
func (a *App) getDirectories() []*model.Directory {
    // ...
}
```

### Q: Go 编译错误怎么办？

**症状：**
- `wails build` 或 `wails dev` 时 Go 编译失败
- 提示语法错误或类型错误

**解决方案：**

**清理 Go 缓存：**

```bash
# 清理所有缓存
go clean -cache -modcache -testcache

# 重新下载依赖
go mod tidy

# 重新构建
wails build -clean
```

**常见编译错误：**

1. **未使用的 import**
```go
// ❌ 错误：导入了但未使用
import (
    "os"
    "fmt"
)

// ✅ 解决：删除未使用的 import
import (
    "fmt"
)
```

2. **类型不匹配**
```go
// ❌ 错误：类型不匹配
var path string = 123

// ✅ 解决：使用正确的类型
var path string = "C:\\test"
```

### Q: WebView2 运行时缺失怎么办？

**症状：**
- 应用启动后显示空白窗口
- 提示 WebView2 组件不可用

**解决方案：**

**下载并安装 WebView2 Runtime：**

- 官方下载地址：https://developer.microsoft.com/en-us/microsoft-edge/webview2/
- 选择 "Evergreen Bootstrapper"（在线安装）或 "Standalone Installer"（离线安装）

**检查 WebView2 是否已安装：**

```bash
# 检查 WebView2 运行时版本
reg query "HKLM\SOFTWARE\WOW6432Node\Microsoft\EdgeUpdate\Clients\{F3017226-FE2A-4295-8BDF-00C3A9A7E4C5}" /v pv
```

**注意：** Windows 10/11 通常已经内置 WebView2，无需额外安装。

## 使用问题

### Q: 如何添加工作目录？

**步骤：**

1. 启动 Git Manager 应用
2. 点击顶部工具栏的"添加目录"按钮
3. 填写目录信息：
   - **目录名称：** 自定义显示名称（如"我的工作空间"）
   - **目录路径：** 实际文件系统路径（如 `D:\workspace`）
   - **设为默认：** 是否作为默认启动目录
4. 点击"确定"保存

**注意事项：**
- 路径必须真实存在
- 路径不包含非法字符（`<>:"|?*`）
- 支持相对路径和绝对路径
- 相对路径会自动转换为绝对路径

---

### Q: 如何识别 Git 仓库？

**标识：** 文件树中的 Git 仓库文件夹旁边会显示绿色的 ✓ 图标。

**检测规则：**
- 文件夹包含 `.git` 子目录
- 自动递归检测子目录
- Git 仓库内部的子目录不会再次标记

**示例：**
```
📁 workspace_ai
  📁 git-manager  [✓] ← Git 仓库根目录
    📁 frontend    ← 普通文件夹
    📁 model       ← 普通文件夹
```

---

### Q: 文件预览支持哪些类型？

**支持的文本文件类型：**
- 文本文件：`.txt`, `.md`, `.markdown`
- 配置文件：`.json`, `.xml`, `.yaml`, `.yml`, `.properties`, `.conf`, `.config`
- 代码文件：`.java`, `.py`, `.js`, `.ts`, `.jsx`, `.tsx`, `.vue`, `.go`, `.rs`, `.c`, `.cpp`, `.h`
- Web文件：`.html`, `.css`, `.scss`, `.less`
- 脚本文件：`.sh`, `.bat`, `.ps1`
- 其他：`.gitignore`, `.env`, `Dockerfile`

**限制：**
- 文件大小不超过 1MB
- 不支持二进制文件（`.exe`, `.png`, `.zip` 等）

---

### Q: 如何克隆远程仓库？

**步骤：**

1. 在文件树中选择目标父目录
2. 右键点击目录（如果已实现右键菜单）或使用顶部工具栏
3. 输入远程仓库 URL：
   - HTTPS：`https://github.com/user/repo.git`
   - SSH：`git@github.com:user/repo.git`
4. 点击"克隆"按钮

**注意事项：**
- 目标目录不能已存在同名文件夹
- 需要配置 Git 认证（HTTPS 凭据助手或 SSH 密钥）
- 大型仓库克隆可能需要较长时间

## 性能问题

### Q: 文件树加载缓慢？

**可能原因：**
- 目录包含大量文件/文件夹
- 深层嵌套的目录结构
- 网络驱动器访问慢

**解决方案：**

1. **使用懒加载**（已实现）
   - 仅展开时加载子节点
   - 避免一次性加载整个树

2. **避免添加过大的工作目录**
   - 不要添加整个 C 盘根目录
   - 仅添加实际需要的工作空间

3. **优化 .gitignore**
   - 排除不必要的文件夹（node_modules, build 等）

---

### Q: 应用启动慢？

**可能原因：**
- 工作目录配置文件过大
- 默认工作目录在网络驱动器上
- WebView2 首次初始化

**解决方案：**

1. **检查 `data/directories.json` 文件大小**
   - 应该小于 10KB
   - 如果过大，删除不需要的目录配置

2. **设置本地默认目录**
   - 避免默认目录在网络驱动器上

3. **首次启动较慢是正常的**
   - WebView2 需要初始化
   - 后续启动会更快

## 调试技巧

### Q: 如何查看前端日志？

**开发模式：**

1. 打开浏览器（应用会自动打开）
2. 按 `F12` 打开开发者工具
3. 切换到 `Console` 标签页
4. 查看日志输出

**注意：** 生产环境（`wails build`）不会输出调试日志。

---

### Q: 如何查看后端日志？

**开发模式：**

- 运行 `wails dev` 的终端窗口
- `println()` 输出会显示在终端

**生产模式：**

- 目前不支持日志文件
- 需要日志功能，请参考 [开发规范](./开发规范.md)

---

### Q: 如何调试 Wails 绑定？

**检查绑定是否正确生成：**

```bash
# 查看 wailsjs 目录
ls frontend/wailsjs/go/main/App.js
```

**在前端调用测试：**

```javascript
// 打开浏览器控制台
import { GetDirectories } from '../../wailsjs/go/main/App'

const dirs = await GetDirectories()
console.log('Directories:', dirs)
```

**常见问题：**
- 方法名拼写错误（区分大小写）
- Go 方法未导出（小写字母开头）
- 前端绑定期未生成
